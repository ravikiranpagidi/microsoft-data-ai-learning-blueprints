# Learning Resources

## Concept

Learning Fabric requires both official documentation and hands-on practice. This repo gives a guided practice project and links to official references.

## Why It Matters

Fabric changes over time. Official docs should be the source of truth for service limits, feature availability, and supported deployment behavior.

## How It Works in Microsoft Fabric

Microsoft Learn provides documentation for Fabric workloads, OneLake, Lakehouse, Data Factory, Git integration, deployment pipelines, and icons.

## Real-World Use Case

Use the Retail Banking Customer Analytics project as your practice implementation while reading official Fabric docs alongside it.

## Beginner Explanation

Do not try to memorize everything first. Build a small end-to-end flow and revisit concepts when they become relevant.

## Enterprise Best Practice

Use official docs for current platform behavior and this repo for architecture practice.

## Common Mistakes

- Relying only on videos without building anything.
- Learning Power BI consumption without understanding data engineering layers.
- Using old blog posts as the only source for current feature behavior.
- Skipping governance until after the model is published.

## Practical Recommendation

Pair each learning session with a small hands-on task from the roadmap.
