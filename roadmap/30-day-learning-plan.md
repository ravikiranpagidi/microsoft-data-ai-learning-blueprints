# 30-Day Learning Plan for Beginners

This plan helps a beginner build confidence with Microsoft Fabric Data Engineering in one month.

## Week 1: Understand the Platform

Day 1: Read the README and repository overview.

Day 2: Learn what Microsoft Fabric is and how workspaces are organized.

Day 3: Learn OneLake and Lakehouse basics.

Day 4: Review Lakehouse vs Warehouse decisions.

Day 5: Review Notebook vs Pipeline decisions.

Day 6: Study the Retail Banking sample data.

Day 7: Explain the end-to-end architecture out loud.

## Week 2: Build the Lakehouse Flow

Day 8: Create a Fabric workspace.

Day 9: Create a Lakehouse.

Day 10: Upload sample CSV files.

Day 11: Run setup and Bronze ingestion notebooks.

Day 12: Inspect Bronze tables and metadata.

Day 13: Run Silver transformation.

Day 14: Validate row counts and key checks.

## Week 3: Build Business-Ready Data

Day 15: Learn star schema basics.

Day 16: Run the Gold dimensional model notebook.

Day 17: Inspect dimensions and fact table.

Day 18: Run data quality checks.

Day 19: Run SQL views in the SQL analytics endpoint.

Day 20: Query business metrics.

Day 21: Explain how Gold differs from Silver.

## Week 4: Connect Power BI and Think Like an Engineer

Day 22: Review semantic model design.

Day 23: Create relationships in Power BI.

Day 24: Add core measures.

Day 25: Build a transaction trend page.

Day 26: Build a customer segment page.

Day 27: Review governance and PII handling.

Day 28: Review CI/CD basics.

Day 29: Answer interview guide questions.

Day 30: Present the solution as if you were explaining it to a project team.

## Completion Outcome

By the end of 30 days, you should be able to build and explain a complete beginner-level Fabric lakehouse project from source files to Power BI.
