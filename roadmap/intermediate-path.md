# Intermediate Path

## Goal

Move from a working demo to a reliable team pattern.

## Learning Steps

1. Build a Fabric Data Pipeline for ingestion.
2. Parameterize notebook execution.
3. Add data quality checks.
4. Understand dimensional modeling.
5. Create reusable SQL endpoint views.
6. Apply security basics and PII classification.
7. Review performance and cost considerations.
8. Create a shared Power BI semantic model.

## Success Criteria

- You can explain pipeline vs notebook responsibilities.
- You can add a new data quality rule.
- You can model a fact and dimension relationship.
- You can describe how a semantic model should consume Gold data.
