# 30-Day Learning Plan

> **Learning stage:** Decision, career, and community
> **Blueprint anchor:** Retail Banking Customer Analytics on Microsoft Fabric
> **Outcome:** Apply 30-day learning plan to the Retail Banking Customer Analytics blueprint.
> **Navigate:** [Interview Preparation Guide](Interview-Preparation-Guide) | [Home](Home) | [90-Day Professional Growth Plan](90-Day-Professional-Growth-Plan)
> **Quick links:** [FAQ](FAQ) | [Glossary](Glossary) | [Contributor Guide](Contributor-Guide)

## Purpose

Provide a practical 30-day beginner learning plan for Microsoft Fabric Data Engineering using this repo.

## Who Should Read This

Beginners, students, Power BI developers, and Azure data professionals who want a structured first month.

## Plan Overview

This plan assumes 45 to 90 minutes per day. If you have less time, treat each day as a session rather than a calendar day.

## Week 1: Fabric Fundamentals

| Day | Focus | Hands-on task | Expected outcome |
| ---: | --- | --- | --- |
| 1 | What is Fabric? | Read Home and Fabric Fundamentals. | You can explain Fabric workloads. |
| 2 | OneLake | Read OneLake Explained. | You understand shared storage and shortcuts. |
| 3 | Lakehouse | Read Lakehouse Concepts and Files vs Tables. | You can explain Files and Tables. |
| 4 | Lakehouse vs Warehouse | Read decision pages. | You can choose the right item for a scenario. |
| 5 | Pipelines vs Notebooks | Read orchestration pages. | You understand tool responsibilities. |
| 6 | Repo orientation | Explore folders and README. | You know where notebooks, SQL, and docs live. |
| 7 | Review | Write a one-page summary. | You can explain the platform to someone else. |

## Week 2: Lakehouse and Notebooks

| Day | Focus | Hands-on task | Expected outcome |
| ---: | --- | --- | --- |
| 8 | Workspace setup | Create workspace and Lakehouse. | Environment is ready. |
| 9 | Sample data | Upload CSV files. | Source files are in the Files area. |
| 10 | Setup notebook | Run notebook 00. | Paths and readiness are validated. |
| 11 | Bronze | Run notebook 01. | Bronze tables and audit exist. |
| 12 | Silver | Run notebook 02. | Cleaned Silver tables exist. |
| 13 | Transformation review | Inspect code and row counts. | You understand cleaning and deduplication. |
| 14 | Troubleshooting | Fix one intentional path or file issue. | You can debug beginner problems. |

## Week 3: Medallion Architecture and Dimensional Modeling

| Day | Focus | Hands-on task | Expected outcome |
| ---: | --- | --- | --- |
| 15 | Medallion architecture | Read Bronze, Silver, Gold pages. | You can explain layer responsibilities. |
| 16 | Gold modeling | Run notebook 03. | Dimensions and fact table exist. |
| 17 | Fact grain | Explain fact_transaction grain. | You can discuss model design. |
| 18 | SQL endpoint | Run create_gold_views.sql. | SQL views exist. |
| 19 | Validation | Run validation_queries.sql. | You can reconcile counts and keys. |
| 20 | Business metrics | Run business_metrics.sql. | You can answer business questions. |
| 21 | Review | Draw the model on paper. | You can explain relationships. |

## Week 4: Power BI, Governance, CI/CD Basics

| Day | Focus | Hands-on task | Expected outcome |
| ---: | --- | --- | --- |
| 22 | Power BI consumption | Read Power BI and Semantic Model pages. | You know the consumption pattern. |
| 23 | Power BI-ready views | Run notebook 06 and SQL views. | You can inspect reporting shapes. |
| 24 | Data quality | Run notebook 04. | DQ results are created. |
| 25 | Optimization | Run notebook 05. | You understand table health checks. |
| 26 | Governance | Read governance, access, PII pages. | You can describe least privilege. |
| 27 | CI/CD | Read CI/CD and Dev/Test/Prod pages. | You understand promotion strategy. |
| 28 | Common mistakes | Review mistakes page. | You can identify project risks. |
| 29 | Interview practice | Answer 10 interview questions. | You can speak confidently. |
| 30 | Portfolio wrap-up | Write a LinkedIn or blog summary. | You have a shareable learning artifact. |

## Best Practices

- Keep notes as you learn.
- Re-run notebooks after changing sample data.
- Explain every table in plain English.
- Practice both hands-on execution and architecture explanation.

## Weekly Demo Milestones

| Week | Demo you should be able to give | Evidence from repo |
| --- | --- | --- |
| Week 1 | Explain Fabric, OneLake, Lakehouse, and tool choices | Wiki notes and decision tables |
| Week 2 | Show CSV files becoming Bronze and Silver tables | Notebooks 00-02 and row counts |
| Week 3 | Show Gold star schema and SQL metrics | Notebook 03 and SQL scripts |
| Week 4 | Explain Power BI, governance, DQ, and CI/CD readiness | Notebook 04-06, governance docs, CI/CD docs |

## Beginner Reflection Questions

At the end of each week, answer these in your own words:

- What changed in my understanding of Fabric this week?
- Which artifact did I actually run or inspect?
- What mistake would I avoid in a real project?
- Which interview question can I now answer better?
- What would I show in a portfolio demo?
