# Contributing

Thank you for helping improve the Microsoft Fabric Real-Time Intelligence Blueprint.

This blueprint is designed for practical learning and architecture discussion. Contributions should help learners, practitioners, architects, and teams understand how to build real-time analytics solutions with Microsoft Fabric.

## Good Contributions

- Add KQL queries for operational monitoring.
- Improve event generator realism.
- Add new event schemas or logistics scenarios.
- Add Real-Time Dashboard layout examples.
- Add Fabric Activator alert patterns.
- Add Lakehouse or Power BI implementation examples.
- Improve governance, security, cost, or troubleshooting guidance.
- Fix broken links, typos, or unclear explanations.

## Standards

- Keep examples synthetic and safe for public use.
- Do not commit secrets, tenant IDs, connection strings, or production data.
- Use clear names for tables, fields, and files.
- Prefer small pull requests with a clear learning outcome.
- Add comments where a beginner would need context.
- Cite official Microsoft documentation when describing platform behavior.

## Validation

Before opening a pull request, check what applies:

- Python files compile.
- JSON schemas parse.
- KQL files are readable and grouped by use case.
- Markdown links point to the correct topic folder.
- Sample events do not contain real customer or operational data.

For repository-wide guidance, read [../CONTRIBUTING.md](../CONTRIBUTING.md).
