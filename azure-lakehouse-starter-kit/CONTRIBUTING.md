# Contributing

Thank you for helping improve the Azure Lakehouse Starter Kit.

This blueprint is designed for practical Azure Data Engineering learning and implementation planning. Contributions should improve clarity, technical quality, enterprise usefulness, or hands-on learning value.

## Good Contributions

- Improve ADF pipeline examples.
- Add Databricks notebook patterns.
- Add Delta Lake SQL examples.
- Add data quality rules and tests.
- Add Unity Catalog, Purview, or Key Vault guidance.
- Add Power BI semantic model examples.
- Add CI/CD improvements.
- Fix broken links, typos, or unclear explanations.

## Standards

- Use fictional data only.
- Do not commit secrets, tenant IDs, subscription IDs, or production paths.
- Keep examples readable for beginners.
- Keep code realistic enough for practitioners.
- Cite official Microsoft or Databricks documentation when describing platform behavior.

For repository-wide guidance, read [../CONTRIBUTING.md](../CONTRIBUTING.md).
