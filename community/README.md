## Ways Contributors Can Extend the Repo

- Add incremental loading examples.
- Add a Power BI template report.
- Add screenshots from Fabric implementation steps.
- Add Dataflow Gen2 examples.
- Add monitoring and alerting patterns.
- Add more interview scenarios.
- Add another domain such as healthcare, insurance, or retail sales.

## Contribution Lanes

| Lane | What to contribute | Good first issue examples |
| --- | --- | --- |
| Beginner learning | Simpler explanations, diagrams, glossary additions | Clarify OneLake vs Lakehouse |
| Engineering examples | Notebooks, SQL, data quality, pipeline patterns | Add a late-arriving data example |
| Enterprise readiness | Governance, CI/CD, access, release checklists | Add a production support runbook |
| Power BI | Semantic model guidance, DAX, report wireframes | Add a dashboard page layout |
| Community content | Blog posts, videos, meetup demos | Add a workshop agenda |

## Maintainer Review Principles

- Practical beats theoretical.
- Official Microsoft documentation should be cited for platform behavior.
- Synthetic examples only.
- Beginner-friendly explanations are welcome even when the concept is advanced.
- Enterprise guidance should include ownership, security, and operational trade-offs.
