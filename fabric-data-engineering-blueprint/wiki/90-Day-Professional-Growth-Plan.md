# 90-Day Professional Growth Plan

> **Learning stage:** Decision, career, and community
> **Blueprint anchor:** Retail Banking Customer Analytics on Microsoft Fabric
> **Outcome:** Apply 90-day professional growth plan to the Retail Banking Customer Analytics blueprint.
> **Navigate:** [30-Day Learning Plan](30-Day-Learning-Plan) | [Home](Home) | [Contributor Guide](Contributor-Guide)
> **Quick links:** [FAQ](FAQ) | [Glossary](Glossary) | [Contributor Guide](Contributor-Guide)

## Purpose

Provide a 90-day professional growth plan for becoming job-ready in Microsoft Fabric Data Engineering.

## Who Should Read This

Working professionals, Azure data engineers, Power BI developers, and career switchers building portfolio depth.

## Plan Overview

The 90-day plan moves from core skills to real-world implementation and then enterprise readiness. The goal is not only to run a demo. The goal is to explain, extend, govern, and present a Fabric data engineering solution.

## Month 1: Core Fabric Skills

### Skills to Learn

- Fabric workloads and terminology.
- OneLake, Lakehouse, Files, and Tables.
- Data Pipelines, Notebooks, and Dataflow Gen2.
- PySpark basics for cleaning and joining data.
- Delta tables and medallion architecture.

### Mini Projects

- Run the Retail Banking blueprint end to end.
- Add five new rows to each sample CSV and rerun the pipeline.
- Add one new DQ rule and explain the result.
- Create a diagram of the architecture.

### Portfolio Tasks

- Publish a short article: What I learned building Bronze, Silver, and Gold in Fabric.
- Record a three-minute demo walkthrough.

## Month 2: Real-World Implementation

### Skills to Learn

- Dimensional modeling.
- SQL Analytics Endpoint views.
- Power BI semantic model design.
- Data quality design.
- Incremental load concepts.
- Operational monitoring basics.

### Mini Projects

- Extend the sample with a new entity such as customer_interactions.
- Build a new Gold fact or dimension.
- Add Power BI measures and a report mockup.
- Create validation queries for the new entity.

### Interview Preparation

Practice explaining:

- Lakehouse vs Warehouse.
- Notebook vs Pipeline.
- Gold table vs SQL view.
- How you validate business keys.
- How you handle PII.

### Community Contribution Ideas

- Improve a Wiki page.
- Add a new interview question.
- Add a new SQL business metric.
- Improve a diagram or checklist.

## Month 3: Enterprise Readiness

### Skills to Learn

- CI/CD and deployment pipelines.
- Dev/Test/Prod workspace strategy.
- Access control and governance.
- ADR writing.
- Performance and optimization.
- Data product architecture.

### Mini Projects

- Create a release checklist for a new version.
- Write an ADR for a design choice.
- Create a Dev/Test/Prod promotion plan.
- Add monitoring SQL or notebook examples.
- Design an enterprise pattern for a second domain.

### Portfolio-Building Tasks

- Publish a 5-part blog series based on the repo.
- Present the architecture at a meetup or study group.
- Create a GitHub issue and pull request contribution.
- Prepare a 10-minute interview project walkthrough.

## 90-Day Outcome

By the end, you should be able to build a Fabric Lakehouse project, explain design decisions, apply data quality, discuss governance, plan CI/CD, and present the work as a portfolio-ready project.

## Professional Capability Targets

| Capability | Month 1 | Month 2 | Month 3 |
| --- | --- | --- | --- |
| Build | Run the provided notebooks | Extend the model with one new entity | Design a second domain pattern |
| Explain | Define core Fabric concepts | Explain Gold modeling and metrics | Defend architecture choices with ADRs |
| Govern | Identify PII and DQ rules | Add quality checks and validation queries | Propose access model and release gates |
| Deliver | Run locally in a workspace | Create a demo report or semantic model outline | Prepare Dev/Test/Prod and CI/CD story |
| Contribute | Fix a doc typo or clarify a page | Add an example query or question | Open a polished PR with tests/docs |

## Portfolio Evidence Checklist

- [ ] Architecture diagram with Fabric items and data flow.
- [ ] Screenshots or notes from notebook execution.
- [ ] SQL validation and business metrics examples.
- [ ] Data quality rules and report output.
- [ ] Semantic model measure list.
- [ ] Governance and access-control explanation.
- [ ] ADR explaining one design decision.
- [ ] Short demo script for interviews or LinkedIn.
