# 90-Day Fabric Data Engineer Growth Plan

This plan is for professionals who want to grow from basic Fabric usage to production-ready data engineering patterns.

## Days 1-30: Foundation and End-to-End Build

Focus areas:

- Fabric workspace concepts.
- OneLake and Lakehouse.
- Medallion architecture.
- Pipelines and notebooks.
- Delta tables.
- SQL analytics endpoint.
- Power BI semantic model basics.

Deliverables:

- Run the full Retail Banking blueprint.
- Explain every layer and table.
- Build a simple Power BI report.
- Complete the 30-day beginner plan.

## Days 31-60: Engineering Depth

Focus areas:

- Parameterized notebooks.
- Data quality automation.
- Row count reconciliation.
- Dimensional modeling.
- Incremental load patterns.
- Direct Lake vs Import decisions.
- Performance and cost basics.

Deliverables:

- Add a new source entity.
- Add three new data quality rules.
- Create a new SQL metrics view.
- Create a release note for a model change.
- Explain when to choose Lakehouse, Warehouse, Dataflow Gen2, Notebook, Shortcut, Copy, Direct Lake, and Import.

## Days 61-90: Enterprise Readiness

Focus areas:

- Dev/Test/Prod workspaces.
- Git integration and deployment pipelines.
- Governance and access control.
- PII handling.
- Data product ownership.
- Monitoring and operational readiness.
- Community contribution and documentation.

Deliverables:

- Design a Dev/Test/Prod promotion flow.
- Complete the Fabric Project Checklist.
- Create an Architecture Decision Record.
- Define a data owner/steward model.
- Present an enterprise architecture pattern.
- Submit a documentation, SQL, or notebook contribution to this repo.

## Professional Growth Outcomes

After 90 days, you should be able to:

- Build a production-minded Fabric lakehouse solution.
- Explain major architecture trade-offs.
- Design a governed semantic model.
- Add quality gates before business consumption.
- Participate in enterprise Fabric design reviews.
- Mentor beginners using this repository.
